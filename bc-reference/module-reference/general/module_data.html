{module_data}

Description

{module_data} allows access to data returned by v3 API endpoints.

To learn more about the {module_data} please take a look at the Consuming APIs in the front-end using module_data article

The sorting, filtering, ordering and pagination capabilities are identical to the v3 APIs. The only difference is when writing the module's syntax curly and square braces need to be escaped using a slash character, see the How to section for code samples.


Syntax

{module_data version="v3" collection="MyCustomers" resource="customers" skip="0" limit="55" order="firstName" resourceId=""}

Parameters

  • resource - This parameter is used to specify the name of the resource which is going to be queried.
  • version - This parameter is used to specify the version of the resource which is going to be queried. The value should be "v3"
  • collection - The name of the Liquid collection the result will be placed in.
  • skip - This parameter specifies how many records to be skipped (the offset from the beginning of the collection). Take a look at the API v3: Sorting, filtering and partial representation methods article for more details on this parameter.
  • limit - This parameter specifies how many items to be included from the total number of items returned by the module. Take a look at the API v3: Sorting, filtering and partial representation methods article for more details on this parameter.
  • where - This parameter is used to add a filter to the module. Take a look at the API v3: Sorting, filtering and partial representation methods article for more details on this parameter.
  • order - This parameter specifies the ordering criteria. Take a look at the API v3: Sorting, filtering and partial representation methods article for more details on this parameter.
  • fields - This parameter is used to add additional fields (which might be hidden by default).
  • resourceId - This parameter indicates that a specific item id must be fetched from the specified resource collection.
  • subresource - This parameter specifies the projection which must be queried for a given resource item.
  • template - This parameter specifies if the module output layout (html) is get from current page (inline) or from a custom template (template="" vs. template="[custom_templatefile_path]")

Examples

  • {module_data resource="securezones" resourceId="51" subresource="customers" version="v3" collection="secureZoneList"} - get the list of customers subscribed to the secure zone with the ID 51. Take a look at the How to section for more samples and code snippets.

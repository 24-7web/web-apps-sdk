Pages and secure zones

    List the pages in a particular secure zone

    Request

    • Method: GET
    • Version: v3, this API call supports additional sort and filter mechanisms as well as retrieving only a sub-set of fields
    • Server: https://[app key here]-[site_ID here]-apps.worldsecuresystems.com. Take a look at the OAuth in Business Catalyst document for more info on how this URL is formed.
      • Note: When building apps always use relative request URLs. Do not use the "full" URL above because you might have problems running your app on a different site as the site_ID parameter will be different.
    • Path: /webresources/api/v3/admin/sites/current/securezones/[secure_zone_ID]/pages
      • Alternatively, use use siteID instead of 'current'
    • Authorization header: This should contain the authorization token. Here is how to obtain the token.

    Response

    An array of the first 100 web pages contained by the secure zone.

    Sample code

    var request = $.ajax({
    url: "/webresources/api/v3/sites/current/securezones/3309/pages",
    type: "GET",
    conenection: "keep-alive",
    contentType: "application/json",
    headers: {
        "Authorization": $.cookie('access_token')
    }
});
request.done(function (msg) {
    console.log(msg);
})
request.fail(function (jqXHR) {
    console.log("Request failed.");
    console.log("Error code: " + jqXHR.status);
    console.log("Error text: " + jqXHR.statusText);
    console.log("Response text: " + jqXHR.responseText);
})

    Add pages to a secure zone

    Request

    • Method: POST
    • Version: v3
    • Server: https://[app key here]-[site_ID here]-apps.worldsecuresystems.com. Take a look at the OAuth in Business Catalyst document for more info on how this URL is formed.
      • Note: When building apps always use relative request URLs. Do not use the "full" URL above because you might have problems running your app on a different site as the site_ID parameter will be different.
    • Path: /webresources/api/v3/admin/sites/current/[secure_zone_ID]/pages
      • Alternatively, use use siteID instead of 'current'
    • Authorization header: This should contain the authorization token. Here is how to obtain the token.

    Response

    The response should be "204 no content" if the page was successfully added to the secure zone.

    Sample code

    var request = $.ajax({
     url: "/webresources/api/v3/sites/current/securezones/3309/pages",
      data: JSON.stringify({
          "items": [{
            "objectId": 4518424
          }]
        }),
    type: "POST",
    contentType: "application/json",
    headers: {
        "Authorization": $.cookie('access_token')
    }
});
request.done(function (msg) {
    console.log(msg);
})
request.fail(function (jqXHR) {
    console.log("Request failed.");
    console.log("Error code: " + jqXHR.status);
    console.log("Error text: " + jqXHR.statusText);
    console.log("Response text: " + jqXHR.responseText);
})

    Remove pages from a secure zone

    Request

    • Method: DELETE
    • Version: v3
    • Server: https://[app key here]-[site_ID here]-apps.worldsecuresystems.com. Take a look at the OAuth in Business Catalyst document for more info on how this URL is formed.
      • Note: When building apps always use relative request URLs. Do not use the "full" URL above because you might have problems running your app on a different site as the site_ID parameter will be different.a
    • Path: /webresources/api/v3/admin/sites/current/securezones/[secure_zone_ID]/pages?items=[page1_ID,page2_ID,page3_ID]
      • Alternatively, use use siteID instead of 'current'
    • Authorization header: This should contain the authorization token. Here is how to obtain the token.

    Response

    The response should be "204 no content" if the pages were successfully removed from the the secure zone.

    Sample code

    var request = $.ajax({
    url: "/webresources/api/v3/sites/current/securezones/3309/pages?items=[4518424]",
    type: "DELETE",
    contentType: "application/json",
    headers: {
        "Authorization": $.cookie('access_token')
    }
});
request.done(function (msg) {
    console.log(msg);
})
request.fail(function (jqXHR) {
    console.log("Request failed.");
    console.log("Error code: " + jqXHR.status);
    console.log("Error text: " + jqXHR.statusText);
    console.log("Response text: " + jqXHR.responseText);
})

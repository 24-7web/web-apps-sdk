Product list layout customizations

In this article we will take a look at how we can replace the product list layout using module_data. The main advantage of this setup is you are not limited to the list of tags available on the Product List layout.

Description

To implement this setup we will need to go in the Overall shop layout and replace the product listing outputted by {tag_productlist} with module_data.

Once module_data is in palce we will use a custom layout to list the products.

The build

Step 1 - go in the Overall shop layout and replace tag_productlist with module_data like so:

{module_data resource="catalogs" version="v3" subresource="products" resourceId="{{this.globals.get.CatalogueID}}" template="/demo/product-list.tpl"}

This instance of module_data lists all the products in the current catalog (just like tag_productlist) and renders the products using the product-list.tpl custom template

Step 2 - create the tag_productlist template and paste the code below:

{% for item in items %}
	{module_data resource="products" version="v3" subresource="prices" resourceId="{{item.product.id}}" collection="prices"}
	<div class="product-item">
		{% assign ppl = item.product.poplets | split: ";" %}
		<ul>
			<li><img src="{{ppl[0]}}?Action=thumbnail&amp;Width=600&amp;Height=900&amp;algorithm=fill_proportional" alt="{{item.product.name}} {{item.product.name}}"/></li>
			<li><img src="{{ppl[1]}}?Action=thumbnail&amp;Width=600&amp;Height=900&amp;algorithm=fill_proportional" alt="{{item.product.name}} {{item.product.name}}"/></li>
		</ul>
		<h4>{{item.product.name}}</h4>
		<p>â‚¬{{prices.items[0].salePrice}}</p>
	</div>
{% endfor %}

Good practise when using module_data is to always use {{this | json}} to explore the objects and see whay they contain.

By default when using this implementation the product prices will not be available in our custom template, we will need to explicitly add them to the layout so we can access them. To do this we are using the module_data at line 2 of the custom template.

Because the module_data that brings prices to the module is placed inside the FOR loop it is executed for each product in the list and places the price object in the prices collection. We can now render the price for each product using prices.items[0].salePrice
